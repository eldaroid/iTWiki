
## Хранение данных в браузере: кэш браузера


Cookie:
- хранится в браузере и на сервере
- отправляются по запросу
- Объем: до 4 кб
- хранится: по дефолту в сессии, можно установить срок хранения
- доступно из: любого окна (вкладки)


- данные сохраняются после перезагрузки страницы;
- данные привязаны к конкретному домену;
- могут управляться сервером;
- количество записей ограничено ~20 шт;
- каждая запись хранит не более 4 Кб данных;
- для чтения, добавления, изменения и удаления записей нужны кастомные утилитарные функции.


Из-за того что куки пересылаются с каждым запросом, они могут ухудшать производительность (особенно при использовании мобильных сетей). 

В качестве хранилищ данных на стороне пользователя вместо них можно использовать [Web storage API](https://developer.mozilla.org/ru/docs/Web/API/Web_Storage_API) (localStorage и sessionStorage) и [IndexedDB](https://developer.mozilla.org/ru/docs/Web/API/IndexedDB_API).

### LocalStorage:
- хранятся в браузере
- отправляются ?
- Объем: до 4 кб
- хранятся: данные не имеют сроки давности и хранятся всегда, пока не очистить кэш браузера  
- доступно из: любого окна (вкладки)

```js
// записываем переменную nameStorage
const name = 'Mario Dev';
localStorage setItem('nameStorage', JSON.stringify(name));
// достаем переменную nameStorage
const getName = localStorage.getItem('nameStorage');
console.log(getName) ;
```

- данные сохраняются после перезагрузки страницы;
- данные привязаны к конкретному домену;
- не могут управляться сервером;
- количество и объём записей ограничен 5 Мб данных;
- для чтения, добавления, изменения и удаления записей есть встроенные методы.

### SessionStorage:
- хранятся в браузере
- отправляются ?
- Объем: до 4 кб
- хранятся: пока не закроется вкладка 
- доступно из: внутри одной вкладки

```js
// записываем переменную nameStorage
const name = 'Mario Dev';
sessionStorage setItem('nameStorage', JSON.stringify(name));
// достаем переменную nameStorage
const getName = sessionStorage.getItem('nameStorage');
console.log(getName) ;
```
