## Cookie

1. [HTTP-куки](https://developer.mozilla.org/ru/docs/Web/HTTP/Guides/Cookies)

Cookie — это небольшие фрагменты данных (до 4 КБ на один cookie), которые браузер сохраняет на компьютере пользователя и автоматически передает на сервер в HTTP-заголовке Cookie при каждом запросе к соответствующему домену. Сервер может запросить установку cookie через заголовок `Set-Cookie`. Является частью HTTP-протокола, определённого в спецификации [RFC 6265](https://datatracker.ietf.org/doc/html/rfc6265).

Свойства:
* Cookie в основном используются back-end для сессий, авторизации и трекинга; front-end может читать и писать их через document.cookie (если не установлен флаг HttpOnly);
* Cookie имеют срок годности (Max-Age/Expires, по дефолту сессионный срок: удаляется при закрытии браузера). После истечения срока браузер удаляет их и перестает передавать в запросах.

Применяется для сохранения данных на стороне пользователя, на практике обычно используется для:
* Аутентификации пользователя;
* Хранения персональных предпочтений и настроек пользователя;
* Отслеживания состояния сеанса доступа пользователя;
* Ведения статистики о пользователях;

### Примеры использования

Н/р, если вход на сайт осуществляется при помощи куки, то после ввода пользователем своих данных на странице входа, куки позволяют:
1) **серверу запомнить**, что **пользователь уже идентифицирован**, и ему разрешён доступ к соответствующим услугам и операциям. 

----

Многие сайты также используют куки для

2) **сохранения настроек пользователя**. Эти настройки могут использоваться для персонализации, которая включает в себя выбор оформления и функциональности. 
* Википедия позволяет авторизованным пользователям выбрать дизайн сайта;
* Поисковая система Google позволяет пользователям выбрать количество результатов поиска, отображаемых на одной странице;

---

Куки также используются для 

3) отслеживания действий пользователей на сайте. Как правило, это делается с целью сбора статистики, а рекламные компании на основе такой статистики формируют анонимные профили пользователей, для более точного нацеливания рекламы.

### Работа кук с сервером

Запрашивая страницу, браузер отправляет веб-серверу короткий текст с HTTP-запросом. Например, для доступа к странице http://www.example.org/index.html, браузер отправляет на сервер `www.example.org` следующий запрос:

браузер -> сервер

`GET /index.html HTTP/1.1 Host: www.example.org`

---

браузер <- сервер

Сервер отвечает, отправляя запрашиваемую страницу вместе с текстом, содержащим HTTP-ответ. Там может содержаться указание браузеру сохранить куки:

`HTTP/1.1 200 OK Content-type: text/html Set-Cookie: name=value (содержимое страницы)`

❗ После создания соokiе все запросы в домене будут автоматически передавать эти cookie. Если при загрузке страницы браузер совершает 30 НТТР запросов, то каждый будет содержать `cookie`. Н/р: АРІ запросы, запросы на картинки, запросы на чтение css и html файлов.

---

браузер -> сервер 

Строка Set-cookie отправляется лишь тогда, когда сервер желает, чтобы браузер сохранил куки. В этом случае, если куки поддерживаются браузером и их приём включён, браузер запоминает строку name=value (имя = значение) и отправляет её обратно серверу с каждым последующим запросом. Например, при запросе следующей страницы `http://www.example.org/spec.html` браузер пошлёт серверу `www.example.org` следующий запрос:

`GET /spec.html HTTP/1.1 Host: www.example.org Cookie: name=value Accept: /`

Этот запрос отличается от первого запроса тем, что содержит строку, которую сервер отправил браузеру ранее. Таким образом, сервер узнает, что этот запрос связан с предыдущим. Сервер отвечает, отправляя запрашиваемую страницу и, возможно, добавив новые куки. Значение куки может быть изменено сервером путём отправления новых строк `Set-Cookie: name=newvalue`. После этого браузер заменяет старое куки с тем же name на новую строку. 


### Работа кук с браузером

Куки также могут устанавливаться программами на языках типа JavaScript, встроенными в текст страниц, или аналогичными скриптами, работающими в браузере. В JavaScript для этого используется объект `document.cookie`. Например, `document.cookie = "temperature=20"` создаст куки под именем «temperature» и значением 20.
